# 🚀 Fixigo Backend - Render Deployment Guide

This guide describes how to deploy the Fixigo backend to **Render** using a free-tier Web Service.

Because Render's free tier filesystems are ephemeral (erased on restarts/sleeps), **we do not use SQLite for production**. Instead, the backend is configured to connect to your persistent **Supabase PostgreSQL database**.

---

## 📋 Prerequisites

1. A [Render](https://render.com) account.
2. A [GitHub](https://github.com) repository containing this project.
3. Your Supabase PostgreSQL database credentials (which can be found in your local `backend/.env` file).

---

## ⚡ Deployment Method 1: Using Render Blueprints (Recommended)

Render Blueprints use the `render.yaml` file located in the root of the repository to automatically configure the service.

### Step 1: Push Changes to GitHub
Commit and push the new `render.yaml` and this deployment guide to your GitHub repository:
```bash
git add render.yaml backend/DEPLOY_RENDER.md
git commit -m "Add Render deployment configurations"
git push
```

### Step 2: Create a Blueprint Instance on Render
1. Go to the [Render Dashboard](https://dashboard.render.com/).
2. Click **New +** in the top right and select **Blueprint**.
3. Connect your GitHub repository.
4. Render will automatically detect the `render.yaml` file.
5. Fill in the **Service Group Name** (e.g., `fixigo-backend-service`).

### Step 3: Provide Environment Variables
Render will prompt you to enter values for the following environment variables. Copy them from your local `backend/.env` file:
* **`DB_HOST`**: PostgreSQL host (e.g., `aws-1-ap-southeast-2.pooler.supabase.com`)
* **`DB_USER`**: PostgreSQL username (e.g., `postgres.smoluqhnnocohgzvlutc`)
* **`DB_PASSWORD`**: PostgreSQL password
* **`SUPABASE_URL`**: Supabase API URL
* **`SUPABASE_ANON_KEY`**: Supabase Anonymous Key
* **`SUPABASE_SERVICE_KEY`**: Supabase Service Role Key
* **`RAZORPAY_KEY_ID`**: Razorpay Key ID
* **`RAZORPAY_KEY_SECRET`**: Razorpay Key Secret

*Note: `JWT_SECRET` will be automatically generated by Render securely. `DB_USE_SQLITE` is automatically set to `false` in the blueprint configuration.*

### Step 4: Deploy
Click **Apply** (or **Deploy**). Render will spin up your Node.js application.

## 🛠️ Deployment Method 2: Manual Setup on Render Dashboard

If you prefer to configure the service manually through the Render Dashboard:

1. Click **New +** and select **Web Service**.
2. Connect your GitHub repository.
3. Configure the following settings:
   * **Name**: `fixigo-backend`
   * **Language**: `Node`
   * **Root Directory**: `backend` *(Crucial, as the repo root contains Flutter files!)*
   * **Build Command**: `npm install`
   * **Start Command**: `npm start`
4. Expand the **Advanced** section and click **Add Environment Variable**. Add all variables from your `backend/.env` file:
   * Set `DB_USE_SQLITE` to `false`.
   * Add `DB_HOST`, `DB_PORT` (6543), `DB_USER`, `DB_PASSWORD`, `DB_NAME` (postgres), `DB_SSL` (true), `JWT_SECRET`, `NODE_ENV` (production), `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_KEY`, `RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET`.
5. Click **Create Web Service**.

## 🔍 Verification

Once the deployment finishes successfully, Render will provide a public URL (e.g., `https://fixigo-backend.onrender.com`).

### 1. Test Server Health
Open a browser or use `curl` to access the health check endpoint:
```bash
curl https://<YOUR-RENDER-SUBDOMAIN>.onrender.com/health
```
**Expected Response:**
```json
{ "message": "Server is running" }
```

### 2. Verify Supabase Schema
Ensure that you have run the schema in your Supabase SQL editor as per the `SUPABASE_MIGRATION_GUIDE.md`:
1. Open the Supabase Dashboard.
2. Select your project.
3. Open the **SQL Editor**.
4. Paste and execute the contents of the `backend/schema.sql` file.

## 📱 Connecting Flutter App

To connect your Flutter application to the live backend, update your API base URL in your Flutter configuration files to use the new Render URL:

```dart
// Change from localhost to your Render URL:
const String baseUrl = 'https://fixigo-backend.onrender.com';
```
